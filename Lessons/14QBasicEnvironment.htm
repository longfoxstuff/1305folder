Focal Point . The QBasic Environment
Learning . Restrictions:
  • QBasic was written for and bundled with DOS
  • It has all the restrictions of DOS
  • This includes the 8.3 filename structure
  • QBasic does not use a standard GUI
  • QBasic will not recognize network shares (unless they're mapped to a drive letter)
  • It's no longer developed and supported by Microsoft

Benefits:

  • QBasic is SMALL (317KB including the help file)
  • It has great built-in help
  • It has powerful functions and the ability to scale up with additional libraries
  • It is a Great learning language

Starting QBasic:

  • You need to have both qbasic.exe and qbasic.hlp in the same folder
  • At that point you can simply double-click on the executable and QBasic will start up in a window
  • You then have a choice to see the Survival Guide or clear the dialog box
  • Choosing the Survival Guide gives you three choices, the first of which I encourage you to investigate is Using Help

The most helpful key:

  • The Alt key on the keyboard
  • This key will open up the menus
  • If you hit the Alt key by itself you will highlight the File menu
  • You can use the down arrow at this point to open the file menu and go down through the available options
  • If you hit one of the highlighted letters on the other menus instead you will drop down that menu
  • Each menu has highlighted letters for shortcuts
  • The quickest way to leave QBasic is to hit Alt, Down, X
  • Can you see why this is so?

The Windows:

  • There are two windows on the screen by default:  The View window and the Immediate window
  • The View window is where you will write your programs
  • The Immediate window is where you can check variable values and try commands with immediate feedback
  • Remember: QBasic is an Interpreted language
  • f6 switches between the windows

Your first program:

  • Type the following into the View window exactly as I've typed it here:
 
REM My first program
CLS
PRINT "Hello World"
END
  • Hit f5 to run the program
  • What happened?
  • Congratulations!  You've just written your first actual program
  • It doesn't do much but it's a program

Note: "REM" stands for remark. Remarks are ignored when the program is run. It is used for internal documentation within the program.
