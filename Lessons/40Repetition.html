Focal Point . Flow Control - Repetition
Learning .

When we're programming we often need to repeat repetitions.  This is done by implementing Nested Loops.  This is a good description of nested loops from linuxdoc.org:

A nested loop is a loop within a loop, an inner loop within the body of an outer one. What happens is that the first pass of the outer loop triggers the inner loop, which executes to completion. Then the second pass of the outer loop triggers the inner loop again. This repeats until the outer loop finishes. Of course, a break within either the inner or outer loop may interrupt this process.

The following simple program demonstrates the nested loop:

Write a program which will input a number from 1 to 25 (there are 25 lines on a standard DOS screen) and will draw a solid square of asterisks as tall and wide as the number from the user.

REM plong
REM 6-5-2005
REM Program to draw a solid square of asterisks
REM The user inputs a number from 1 to 25 to determine
REM the size of the square

DIM num AS INTEGER
DIM i AS INTEGER, j AS INTEGER

CLS
INPUT "How many stars across and tall do you want your square? ", num

CLS
FOR i = 1 TO num
   FOR j = 1 TO num
      PRINT "*";
   NEXT j
   PRINT
NEXT i

END

Notice that the inner loop is using a variable named "j" which is traditionally the second counter name after "i".

In this program the "i" loop is referring to the number of lines in the square.  The "j" loop is referring to the number of *'s in each line.

The print statement immediately following the "next j" is to tell QBasic to go to the next line since the previous print statement, the one printing the asterisk itself, is ended with a semicolon.  When the "j" loop finishes the program goes on the next value of "i" which is another line.  Hence, the print statement.
