CIST 1305 Lesson 05 Planning Your Program

Lesson Topic PDLC / Methodology Notes
Objectives The CIST 1305 students at the end of this lesson should:
  • Understand the Program Development Life Cycle
  • Have a basic idea how a program works
  
Introduction / Warm-up Introduction to class
  • Review previous work
  • Methodology - organized plan
  
New Information The System Development Life Cycle is not just for software development;  it's for any kind of project development you may encounter.

The Program Development Life Cycle is part of the SDLC;  it's part of the Implementation phase.

This is a bit more detail about the PDLC:

 1. Analyze Problem
  • Review program specifications
  • Meet with analyst and users
  • Identify program components
 2. Design Programs
  • Group activities into modules
  • Devise solution algorithms
  • Test solution algorithms
 3. Code Programs
  • Translate solution algorithms into a programming language
  • Enter program code into computer
 4. Test Programs
  • Remove any syntax errors
  • Remove any logic errors
 5. Formalize Solution
  • Review program code
  • Review documentation
 6. Maintain Programs
  • Identify errors
  • Identify enhancements
  • This step is sometimes included in the center of the cycle because it's so critical to the entire function of the PDLC
 
Key Points Notice that the SDLC nor the PDLC ever ends; the development cycle is continuous.  
Applying New Information Thing about other things that are continuous.
